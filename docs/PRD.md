# PRD: gymzalabin — Notion CMS 활용 운동 로그 대시보드

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | gymzalabin |
| 목적 | Notion을 CMS로 활용한 운동 로그 대시보드 |
| 작성일 | 2026-04-02 |

### 배경 및 목적

Notion API를 CMS로 활용하여 비개발자도 별도 관리자 페이지 없이 Notion에서 직접 운동 기록을 입력·수정할 수 있는 대시보드를 구축한다. 웨이트 트레이닝과 러닝 기록을 시각화하여 트레이닝 흐름을 한눈에 파악할 수 있게 한다.

### CMS로 Notion을 선택한 이유

- Notion에 데이터를 입력하면 웹 대시보드에 즉시 반영
- 별도 관리자 UI 개발 불필요 → 개발 비용 절감
- 비개발자도 익숙한 Notion 인터페이스로 콘텐츠 관리 가능
- Notion API가 무료 제공 범위 내에서 충분히 활용 가능

---

## 2. 주요 기능

### 2.1 운동 기록 시각화

- Notion DB에 웨이트 / 러닝 기록을 입력하면 웹에서 Chart.js 차트로 표시
- 카테고리별(웨이트 / 러닝) 필터링 지원

### 2.2 통계 지표

| 지표 | 설명 |
|------|------|
| 주간 볼륨 | 해당 주의 총 운동량 (무게 × 세트 합산) |
| 최고 기록 (PR) | 종목별 최대 무게 또는 최장 거리 |
| 누적 거리 | 러닝 누적 km |
| 운동 일수 | 기간 내 운동한 날 수 |

### 2.3 운동 기록 테이블

- 날짜 / 종목 / 카테고리 / 루틴 / 무게(kg) / 메모를 테이블로 표시
- 최신순 정렬, 페이지네이션 또는 무한 스크롤

## 4. Notion 데이터베이스 스키마

데이터베이스명: **운동 기록 (Workout Log)**

| 필드명 | Notion 타입 | 설명 | 예시 |
|--------|------------|------|------|
| 이름 | Title | 운동 이름 | 벤치프레스, 5km 러닝 |
| 날짜 | Date | 운동 날짜 | 2026-04-02 |
| 카테고리 | Select | 운동 종류 | 웨이트 / 러닝 |
| 루틴 | Select | 웨이트 부위 또는 러닝 | 가슴 / 등 / 하체 / 어깨 / 팔 / 전신 / - |
| 무게(kg) | Number | 사용 무게 또는 이동 거리(km) | 60, 5.0 |
| 세트 | Number | 세트 수 (웨이트 전용) | 3 |
| 반복 | Number | 반복 횟수 (웨이트 전용) | 10 |
| 메모 | Text | 자유 메모 | 그립 넓게, 페이스 6:00 |

> **스키마 단순화 원칙**: 날짜, 종목, 세트/거리, 메모 4가지 핵심 필드만으로 빠른 입력이 가능하도록 설계한다.

## 5. 화면 구성 (페이지 구조)

```
/                   → 대시보드 메인
  ├─ 통계 카드       → 주간 볼륨 / PR / 누적 거리 / 운동 일수
  ├─ 차트 섹션       → 주간 볼륨 바 차트 / 러닝 거리 라인 차트
  └─ 기록 테이블     → 전체 운동 기록 목록
```

## 8. 제약 및 고려 사항

- **Notion API 요청 제한**: 초당 3회 제한 → ISR 캐싱으로 빈번한 호출 방지
- **공개 여부**: Notion DB는 Integration에만 공유, 외부에 직접 노출 금지
- **데이터 소유**: 모든 데이터는 사용자의 Notion 워크스페이스에 저장
